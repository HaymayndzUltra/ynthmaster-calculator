### 8.2 Core Concepts

**3-Layer Rule Hierarchy:**
1. **Foundation (BIOS)** — Rules 1-2: `alwaysApply: true`. Context discovery + collaboration protocols. Run on EVERY interaction.
2. **Guardians** — Rules 3-6: Activate when relevant. Code quality, modification safety, documentation integrity, rule creation.
3. **Experts** — Common rules: Activate conditionally when task domain matches (UI, accessibility, enterprise).

**Rule Metadata Format (YAML Frontmatter):**
```yaml
---
description: "TAGS: [tag1,tag2] | TRIGGERS: keyword1,keyword2 | SCOPE: scope | DESCRIPTION: One-sentence summary."
alwaysApply: false
---
```

**Directive Grammar:**
- `[STRICT]` — Non-negotiable, mandatory. MUST follow exactly.
- `[GUIDELINE]` — Strong recommendation. Deviation permitted with justification.

**5-Protocol Development Lifecycle:**
```
/bootstrap → /define → /plan → /implement → /review + /retro
   (setup)    (PRD)    (tasks)  (execute)   (audit + learn)
```

**Review System (6 Modes):**
- Code Review (quick)
- Security Check
- Architecture Review
- Design System
- UI/Accessibility
- Pre-Production Security
- Comprehensive (Run All)

**Key Patterns:**
- Context Discovery Protocol (BIOS) runs first, loads relevant rules
- Tool Usage Protocol maps IDE tools to actions
- Context Window Preservation prevents rule decay
- Complex Feature Safety Protocol for high-risk modifications
- Think-First Protocol: plan before execute
- One Parent Task, One Chat rule for context management
- Granular commit strategy with consolidation options
